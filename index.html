我們把社課紀錄整理成一座小型技術部落格。

這裡記錄每週的 AI 工具測試、網頁維護筆記、社團專案更新和作者草稿。 對外訪客只能閱讀公開文章；作者和隊伍維護者可以透過右側面板進入內部功能。

Featured

AI 協助整理資安筆記時，最容易忽略的是「資料放在哪裡」。

社課常把功能做出來當作完成，但真正的網站還要檢查靜態檔、資料檔、登入流程和 API 回傳內容。 這篇文章整理本週維護心得：不要只看畫面，也要看瀏覽器載入了哪些檔案。

作者：社團編輯小組 更新：2026/05/22 分類：Web Security
Frontend

從 HTML 到 JavaScript：公開網站的每個檔案都算公開。

如果機密資訊被放進瀏覽器會下載的檔案，即使畫面沒有顯示，使用者仍然可能從開發者工具看到。 前端可以做體驗，不能當保險箱。

Content Ops

內容更新紀錄也需要審核，不只是文章本身。

自動產生的 JSON、Markdown、sitemap 或 robots 記錄常常被當成小事。 但只要部署到公開網站，它們就和正式頁面一樣會被讀到。

Account

作者登入應該保護內容，而不是只保護按鈕。

登入頁若使用弱密碼或沒有嘗試次數限制，攻擊者可能不需要繞過系統，只要猜中密碼就能看到內部訊息。

Backend

後端有驗證，不代表有授權。

很多漏洞不是「沒登入」，而是「登入了任何帳號就能讀到不屬於自己的資料」。 Supabase、Edge Function 和資料表政策都需要一起檢查。

Drafts

草稿預覽功能常是內容網站的盲點。

未發布文章可能還沒準備好公開，但預覽 API 如果只檢查是否登入，沒有檢查隊伍或作者身分， 就會把草稿內容交給錯的人。

網站狀態

公開文章已同步。內容更新資料由自動化流程產生，維護者需確認沒有把內部筆記一起發布。
